MainActivity.this和this分析

发表于 2017-04-19   |   分类于 Android

相信很多同学在接触this时都有很多困惑，所以今天我们就仔细分析一下MainActivity.this 和 this 的区别

1 . 类名.this 这种语法在什么情况下会出现呢？

答案是 ： 内部类需要用到外部类的成员变量或方法时，并且如果重名则必须“类名.this”指定那个类的成员变量或方法，如果没有重名则该变量可以指定所属类也可以不指定所属类。

见下列：

public class Demo
{
int a = 1;
class A
{
int a = 2;
void yanshi()
{
// 内部类需要用到外部类的成员或方法
System.out.println(“Demo.this.a : “ + Demo.this.a);
}
}
}

我么来分析一下，上面代码中，类A是一个内部类，它想调用外面的a，这里由于a变量在内部类A中和外部类Demo中a重名，所以必须用Demo.this.a来区分调用的是哪一个变量a。

2 . 再看MainActivity.this

public class MainActivity extends Activity implements OnClickListener
{
private Button button;
protected void onCreate(Bundle savedInstanceState)
{
super.onCreate(savedInstanceState);
setContentView(R.layout.activity_main);
button=(Button)findViewById(R.id.button);
button.setOnClickListener(this);
}
public void onClick(View arg0) {}
}

这里我们写入 this ，但是我们知道它的全称是 MainActivity.this

好了，就是这样！
